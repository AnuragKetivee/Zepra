Web API Policy

v2.1

Guidelines and terms for accessing ZepraBrowser Web APIs.

Last Updated: February 2026

This document outlines the policies govering the use of ZepraBrowser's Web APIs. By accessing or using our APIs, you agree to comply with these terms. Ideally, these policies ensure a fair, secure, and privacy-respecting environment for all users and developers.

1. Acceptable Use Policy

ZepraBrowser APIs are designed to enhance user experience, enable automation, and facilitate modern web application development. You agree to use the APIs only for lawful purposes and in accordance with these Terms.

Prohibited Activities

  • ✕ Reverse engineering, decompiling, or disassembling any part of the API.
  • ✕ Using the API to transmit malware, viruses, or harmful code.
  • ✕ Scraping or harvesting user data without explicit consent.
  • ✕ Interfering with or disrupting the integrity or performance of the API services.

2. Rate Limits & Quotas

To ensure stability for all users, we enforce rate limits on API requests. Use of the API is subject to the following quotas unless a separate agreement is in place.

STANDARD TIER 1,000 Requests per hour per IP
AUTHENTICATED TIER 10,000 Requests per hour per Key

Note: Exceeding these limits will result in a 429 Too Many Requests response. Repeated violations may result in temporary or permanent IP bans.

3. Data Privacy & Compliance

We take data privacy seriously. ZepraBrowser is committed to protecting user data and complying with global privacy regulations including GDPR and CCPA.

  • Data Minimization: Only request the data necessary for your application's functionality.
  • Consent: You must obtain explicit user consent before accessing sensitive scopes (e.g., location, microphone).
  • Storage: Data retrieved via the API must be stored securely and deleted when no longer needed.

4. Attribution Requirements

If your application uses ZepraBrowser APIs to display content or data, you must provide appropriate attribution to ZepraBrowser.

  • Z
    Powered by ZepraBrowser

The attribution should be clearly visible and legible to the end-user.

5. Termination of Access

ZepraBrowser reserves the right to suspend or terminate your access to the API at any time, with or without notice, if we believe you are in violation of these terms.

Questions?

If you have questions about this policy or need to report a violation, please contact our developer relations team at api-policy@zeprabrowser.com.
